<Expandable title="what are pipes?">
  pipes are scheduled AI agents that run on your screen data. each pipe is a `.md` file with a prompt and a schedule. an AI agent reads the prompt, queries the screenpipe API, and takes action.

  examples: sync to Obsidian, track time in Toggl, send daily summaries.

  see the [pipes guide](/pipes) for details.
</Expandable>

<Expandable title="what are the hardware requirements?">
  - **minimum**: dual-core CPU, 2GB RAM, 20GB disk
  - **recommended**: quad-core, 4GB+ RAM, 50GB+ SSD
  - runs 24/7 on a MacBook Pro M3 or a $400 Windows laptop
  - ~600 MB RAM, ~10% CPU, ~30 GB/month storage at 1 FPS
</Expandable>

<Expandable title="where is data stored?">
  all data is in `~/.screenpipe/`:
  - `db.sqlite` — metadata, OCR text, transcriptions
  - `data/` — MP4 screen recordings, audio chunks
  - `pipes/` — installed pipes

  to backup, copy `~/.screenpipe/` to another location.
</Expandable>

<Expandable title="what OCR engines are supported?">
  `apple-native` (macOS), `windows-native` (Windows), `tesseract` (Linux), `unstructured` (cloud), `custom`
</Expandable>

<Expandable title="is my data private?">
  yes. everything runs locally. no data leaves your machine unless you explicitly choose a cloud provider (deepgram, unstructured). you control what's captured with `--ignored-windows` and `--included-windows`.
</Expandable>

<Expandable title="how do i connect screenpipe to AI?">
  three ways:
  1. **MCP server** — works with Claude Desktop, Cursor ([guide](/mcp-server))
  2. **pipes** — scheduled AI agents ([guide](/pipes))
  3. **REST API** — `curl http://localhost:3030/search?q=your+query`
</Expandable>

<Expandable title="music sounds bad or keeps pausing with bluetooth headphones">
  this is a macOS bluetooth limitation, not a screenpipe bug. when any app opens a bluetooth microphone, macOS switches the headset from the high-quality A2DP codec (stereo, full bass) to the low-quality HFP codec (phone quality, 8kHz mono). this affects all audio — music, ANC, everything. the same thing happens with Zoom, Google Meet, or any app that uses your bluetooth mic.

  **fix:** go to screenpipe settings → audio devices, and switch the input device from your bluetooth headset to your MacBook's built-in microphone. screenpipe will stop opening the bluetooth mic, macOS stays on the high-quality codec, and your music sounds normal again. screenpipe still captures your voice through the built-in mic.

  note: AirPods don't have this issue — Apple uses a proprietary codec that handles input and output simultaneously without switching profiles.
</Expandable>

<Expandable title="what speech-to-text engines are supported?">
  **local (default)**: `parakeet` (25 languages, fastest), `whisper-large-v3-turbo` (99 languages), `whisper-tiny` (lightweight)

  **cloud**: `deepgram` / `screenpipe-cloud` (highest accuracy)

  on macOS Apple Silicon, parakeet uses Metal GPU acceleration via MLX for 8x faster transcription.
</Expandable>

<Expandable title="how is screenpipe different from Scribe?">
  Scribe creates step-by-step documentation guides from on-demand screen recordings — you press record, walk through a workflow, and it generates an annotated guide. screenpipe is fundamentally different: it captures screen + audio + text continuously and stores everything locally so you (or AI agents) can search your full work history later.

  - **scribe**: cloud-only, on-demand, output is static guides
  - **screenpipe**: local-first, always-on, output is searchable AI memory + REST API

  use scribe for SOPs you want to publish. use screenpipe for AI memory, agent context, or compliance-sensitive captures. see the [side-by-side comparison](https://screenpi.pe/compare/scribe).
</Expandable>

<Expandable title="how is screenpipe different from Guidde?">
  Guidde turns short screen recordings into AI-narrated walkthrough videos with auto-generated voiceover and captions — great for customer education and onboarding videos. screenpipe captures continuously in the background, locally, and gives you AI search across your full work history.

  - **guidde**: cloud-only, on-demand recordings, output is polished videos
  - **screenpipe**: local-first, 24/7 capture, output is structured AI memory you can build on

  use guidde to ship how-to videos. use screenpipe when you need an AI that knows what you did or you're building agents on top of your work data. see the [side-by-side comparison](https://screenpi.pe/compare/guidde).
</Expandable>

<Expandable title="how is screenpipe different from Teramind?">
  Teramind and screenpipe both record screens, keystrokes, and text — same input. they diverge on where the data goes. teramind uploads everything to a centralized dashboard for managers to review (insider threat, productivity scoring, DLP). screenpipe keeps everything local on the employee's device and gives the employee an AI memory of their own work.

  - **teramind**: surveillance model — manager owns the data, real-time DLP rules, closed source
  - **screenpipe**: privacy model — employee owns the data, AI memory + agent substrate, MIT open source

  use teramind if you genuinely need real-time DLP rule enforcement and insider-threat dashboards. use screenpipe for the (much larger) set of buyers who want screen-capture for AI / knowledge / compliance without the surveillance dashboard. screenpipe deploys via Intune/MDM with admin policy controls — same enterprise capabilities, opposite philosophy. see the [side-by-side comparison](https://screenpi.pe/compare/teramind).
</Expandable>
